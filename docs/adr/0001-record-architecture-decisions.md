# ADR 0001: 采用架构决策记录（ADR）

- 状态：已接受
- 日期：2026-07-16

## 背景

三人协同、全员 vibe coding，需要一种轻量方式记录「为什么这么做」，避免决策丢失或反复讨论。

## 决策

采用 ADR（Architecture Decision Records）记录所有重要的架构、技术选型与范围决策，存放于 `docs/adr/`，按序号递增命名（`NNNN-title.md`）。

每条 ADR 包含：背景、决策、后果。影响接口 / 数据模型 / 架构的改动，动手前先补一条 ADR。

## 后果

- 决策可追溯、可评审（随 PR）。
- 新成员可通过 ADR 快速理解历史选择。
