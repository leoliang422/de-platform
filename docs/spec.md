# 数据开发学习 & 面试平台 — 规格说明书 (Spec v1.0)

> 状态：定稿 · 最后更新随 PR 维护
> 本文件是需求与范围的唯一事实来源。任何范围变更需通过 PR 修改本文件，并在 `docs/adr/` 记录决策。

## 1. 概述

面向数据开发方向（数仓 / 大数据）求职者的**学习 + 面试** Web 平台。四大内容板块 + 用户 / 积分 / 支付体系 + 大模型内容加工。

- 内容与界面语言：**纯中文**
- 形态：**仅 Web**（不做原生 App / 小程序）
- 交付程度：**完整功能**（含支付骨架、积分兑换、大模型加工、投稿审核全流程）
- 验收标准：**关键流程端到端可演示**（见第 10 节）

## 2. 目标用户与核心价值

- 用户：准备数据开发岗位面试、或日常学习数仓/大数据知识的求职者与在职者。
- 价值：一站式获取「八股知识 + SQL 练习 + 企业面经 + 实战项目」，并通过贡献内容赚取积分、解锁付费资源。

## 3. 四大内容板块

### 3.1 八股总结
- 按技术域分区（Hive / Spark / Flink / 数仓建模 / 调度 …），每个区可**无限下钻**为子分类。
- 分类结构**动态可扩展**（管理员后台维护）。
- 单条知识可标记为免费 / 付费。

### 3.2 SQL 题库
- 数据开发常见 SQL 题目 + 答案 + 难度 + 标签。

### 3.3 面经
- 按**企业**维度组织，展示岗位、面试经验正文。

### 3.4 项目整理
- 每个项目含：项目描述、项目实现、针对项目的问题与答案讲解。
- 分**免费 / 付费**两级；付费按**单个项目**解锁。
- 项目可动态扩展。

## 4. 用户体系

- 注册 / 登录：**邮箱 + 密码**（bcrypt 加密），JWT（access + refresh）。
- 角色：`user`（普通用户）/ `admin`（管理员）。
- 用户中心：资料、积分余额与账本、已解锁内容、我的投稿及审核状态。

## 5. 积分体系（详见 [points-and-payment.md](points-and-payment.md)）

- 获取：上传内容**审核通过后**发放 —— 面经 +20 / 项目 +100 / SQL +10 / 八股 +10。
- **无每日登录奖励。**
- 消耗：兑换付费内容（按各内容 `price_points`）。
- 全程积分账本可审计，发放幂等（同一投稿只发一次）。

## 6. 支付体系（骨架，详见 [points-and-payment.md](points-and-payment.md)）

- 付费内容**双标价**：现金价 `price_cash` + 积分价 `price_points`，二选一解锁。
- 支付走 `PaymentProvider` 抽象，当前实现 `MockProvider`；真实通道（微信 / 支付宝 / Stripe）后置。
- 解锁后写入 `entitlement` 记录，访问付费内容前校验权限。

## 7. 大模型内容加工

- 用户上传原始经验贴 / 项目 / 题目 → 系统调大模型按统一模板整理为规范 Markdown。
- 提供商：**豆包（火山方舟 Ark，OpenAI 兼容）**，经 `LLMClient` 抽象；无 key 时回退 `MockLLM` 保证可演示。
- 加工为**异步任务**（ARQ + Redis），加工结果进入审核队列。

## 8. 内容录入与审核（两条线并存）

1. **管理员后台**（`/admin`，`admin` 角色）：分类与四类内容的增删改查、审核队列、用户与积分管理、订单查看。
2. **用户投稿流**：用户提交 → 大模型加工 → 人工审核 → 发布并发放积分。

**投稿审核状态机：**

```
draft → processing(LLM 加工中) → pending_review → published
                                     ↘ rejected → (可退回) draft
```

- 仅 `published` 时发放积分，且幂等去重。

- 初始种子数据：提供脚本灌入一批示例内容，便于演示与前端联调。

## 9. 非目标（本期不做）

- ❌ 原生 App / 小程序（仅 Web）
- ❌ 多语言国际化（仅中文）
- ⏳ 真实支付通道（先 Mock，后置）
- 默认不做：评论 / 点赞 / 社区互动、全站搜索与推荐、运营数据看板（如需再评审）

## 10. 验收标准

关键流程端到端可演示：

**注册 → 浏览四大板块 → 投稿 → 大模型加工 → 管理员审核发布 → 获得积分 → 用积分 / 现金(mock) 解锁付费项目 → 查看已解锁内容。**

附：本地一键启动（docker-compose）+ 部署说明。

## 11. 规模假设

- 按小规模设计（数十至数百用户），但分类树、内容、项目均动态可扩展，架构保留水平扩展空间。

## 12. 术语

| 术语 | 含义 |
|---|---|
| 八股 | 高频面试知识点总结 |
| 面经 | 面试经验帖，按企业组织 |
| entitlement | 用户对某付费内容的解锁凭证 |
| ledger | 积分账本，记录每笔增减 |
