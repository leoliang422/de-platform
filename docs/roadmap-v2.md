## M5 · 支付 + 异步队列

### 5A 真实支付（微信 + 支付宝）
现状：`MockProvider` 永远成功；订单 `Order` / 权益 `Entitlement` / 双标价 / 积分扣减闭环已就绪。

- `P0` `PaymentProvider` 落地微信支付（Native 扫码 / JSAPI）与支付宝（当面付 / 电脑网站支付）
- `P0` 统一下单 → 返回支付二维码/跳转链接
- `P0` **异步回调 webhook**：验签 + 幂等 + 订单状态机（pending/paid/failed/refunded/closed）
- `P0` 前端收银台 + 支付结果轮询/回跳页
- `P1` 退款、对账流水、订单超时关闭
- 完成标准：沙箱支付成功 → 回调 → 订单 `paid` → `entitlement` 自动发放；重复回调不重复发放。

### 5B 异步队列
现状：ARQ + Redis Worker 代码已在，默认同步。

- `P0` 接 Upstash Redis；Render 起独立 Worker 服务；`TASK_QUEUE_ENABLED=true`
- `P0` 投稿加工转异步（提交即 `processing`，Worker 加工后更新）
- `P1` 失败重试 + 死信；任务：投稿加工、邮件、支付回调后处理、通知扇出
- 完成标准：投稿提交立即返回，Worker 异步完成并可查询状态。

## M6 · 社区地基（账号安全 + 富媒体 + 通知）

私聊/社区强依赖以下能力，先行补齐。

### 账号与安全
- `P0` 用户/管理员**修改密码**
- `P0` **找回密码**（邮箱验证码/链接）
- `P0` 个人资料编辑（昵称、**头像**、简介、职业标签）
- `P1` 注册邮箱验证；登录限流防爆破；刷新令牌轮换与登出失效
- `P2` 第三方登录（GitHub / 微信）

### 富媒体（对象存储）
- `P0` 接对象存储（Cloudflare R2 / S3 / 火山 TOS）+ 后端签名直传
- `P0` **图片上传与展示**（内容、头像、聊天图片共用）
- `P1` 本地文件/笔记上传（Markdown / PDF 附件）
- `P0/P1` Markdown 编辑器（代码高亮、粘贴即传图）

### 通知中心
- `P0` 站内通知（审核结果、点赞、评论、**私信**）+ 未读数
- `P1` 邮件通知（Resend 等）；通知偏好设置

## M7 · 私聊 IM + 社交关系（提前）

参考成熟形态：牛客（面经+讨论+私信+内推）、掘金（关注/点赞/评论）、脉脉（职场社交+私信）、知识星球（付费圈子）、Discord（频道实时）。

### 技术选型（已定基调）
- **自建 WebSocket**：FastAPI WebSocket + Redis pub/sub（多实例广播）+ Postgres 消息表。
- 依赖 Render Starter 常驻（已确认）。规模化后再评估托管 IM（融云/环信/腾讯云 IM）。

### 私信功能
- `P0` 一对一私信：文本、图片、发送/送达/已读、未读数
- `P0` **点头像 → 用户主页 → 发起私聊**
- `P0` 会话列表 + 消息历史（分页）+ 消息通知
- `P0` 反骚扰：拉黑、举报、限流、敏感词、"谁可私信我"设置
- `P1` 在线状态、"正在输入"、消息撤回、离线消息补投
- `P2` 群聊 / 话题频道

### 社交关系
- `P1` 关注 / 粉丝、@提及、个人主页（作品/动态）
- `P1` 动态流（关注的人的更新）

### 架构增量
- WebSocket 网关（鉴权握手复用 JWT）
- 消息存储：`conversations` / `messages` / `message_reads` / `blocks`
- 通知扇出走 M5 的队列；内容安全（敏感词/图片审核可选）

## 成本与风险

- **成本**：Render Starter ~$7/月（常驻）；Neon/Upstash/Vercel 免费档起步，按量增长；微信/支付宝有费率与提现规则。
- **风险**：
  - UGC + 私信带来**内容安全与合规**压力（举报/敏感词/审核不可省）。
  - 私聊长连接的可靠性（离线消息、多端、重连）需要投入；MVP 先保证一对一可用。
  - 免费额度天花板（DB 0.5GB、带宽）到量后需升配。

## 执行方式
- 每个里程碑拆成小 PR（延续现有 vibe coding + PR + CI 流程）。
- 每个 P0 子项都要有测试与端到端验证后再合并。
