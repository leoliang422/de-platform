# 积分与付费规则

> 状态：定稿 v1.0 · 数值调整需通过 PR 修改本文件

## 1. 积分获取（审核通过后发放）

| 行为 | 积分 |
|---|---|
| 上传面经（审核通过） | +20 |
| 上传项目（审核通过） | +100 |
| 上传 SQL 题（审核通过） | +10 |
| 上传八股（审核通过） | +10 |

- **无每日登录奖励。**
- 仅当投稿状态流转到 `published` 时发放。
- 发放**幂等**：账本以 `(ref_type, ref_id)` 去重，同一投稿只发一次；审核退回再发布不重复发放。

## 2. 积分消耗

- 用于兑换付费内容，按各内容的 `price_points` 扣除。
- 扣分即在 `point_ledger` 写入负值记录，并生成 `entitlement`。
- 余额不足则拒绝兑换。

## 3. 付费内容与双标价

- 付费内容（付费项目 / 付费八股）**同时设置**：
  - `price_cash`：现金价（单位：元）
  - `price_points`：积分价
- 用户可**二选一**解锁：现金购买 或 积分兑换。

## 4. 解锁流程

### 现金路径（当前为 Mock）
```
下单(order, status=pending)
  → MockProvider 支付成功(status=paid)
  → 写 entitlement(source=purchase)
```

### 积分路径
```
校验积分余额
  → 扣分(point_ledger 负值)
  → 写 entitlement(source=points)
```

## 5. 访问控制

- 读取付费内容前，校验当前用户是否存在对应 `entitlement`。
- 免费内容任何登录用户可访问（游客是否可读由前端路由与后端策略统一约定，默认需登录）。

## 6. 待定 / 后续

- 真实支付通道（微信 / 支付宝 / Stripe）：经 `PaymentProvider` 接口接入，不改调用方。
- VIP 订阅制：本期不做，若引入将新增订阅态与全站解锁策略（需评审 + ADR）。
- 具体现金定价区间：示例待补（如项目 199 元 / 2000 积分），运营侧确定后填入种子数据。
